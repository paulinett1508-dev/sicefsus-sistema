# 🔧 CORREÇÃO: Bug de Parsing de Valor na Edição de Despesa

## 🐛 PROBLEMA
Ao editar uma despesa, o valor é carregado incorretamente:
- Valor original: R$ 2.699,10
- Após edição: 269,91 (ERRADO!)

## 🎯 CAUSA RAIZ
A função `formatarMoedaInput` trata o número como CENTAVOS quando deveria tratar como REAIS.

---

## ✅ SOLUÇÃO 1: Multiplicar por 100 (RECOMENDADA)

### 📍 Arquivo: `src/components/DespesaForm.jsx`
### 🔍 Localização: Linha ~239 (dentro do useEffect)

### ANTES (ERRADO):
```javascript
valor: despesaParaEditar.valor != null 
  ? formatarMoedaInput(String(despesaParaEditar.valor))
  : "",
```

### DEPOIS (CORRETO):
```javascript
valor: despesaParaEditar.valor != null 
  ? formatarMoedaInput((despesaParaEditar.valor * 100).toFixed(0))
  : "",
```

### 📝 EXPLICAÇÃO:
- `despesaParaEditar.valor` = 2699.10 (número decimal em REAIS)
- `* 100` = 269910 (converte para CENTAVOS)
- `.toFixed(0)` = "269910" (garante string sem decimais)
- `formatarMoedaInput` formata como "2.699,10" ✅

---

## ✅ SOLUÇÃO 2: Formatação Direta (ALTERNATIVA)

Se preferir não depender de `formatarMoedaInput` para centavos:

### ANTES (ERRADO):
```javascript
valor: despesaParaEditar.valor != null 
  ? formatarMoedaInput(String(despesaParaEditar.valor))
  : "",
```

### DEPOIS (CORRETO):
```javascript
valor: despesaParaEditar.valor != null 
  ? despesaParaEditar.valor
      .toLocaleString('pt-BR', { 
        minimumFractionDigits: 2,
        maximumFractionDigits: 2 
      })
      .replace(/\s/g, '') // Remove espaços se houver
  : "",
```

### 📝 EXPLICAÇÃO:
- `toLocaleString('pt-BR')` já formata corretamente como "2.699,10"
- Não precisa converter para centavos
- Mais direto e legível
