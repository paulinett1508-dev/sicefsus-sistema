# 🔒 CORREÇÃO: Bloquear Salvamento com Saldo Insuficiente

## 🚨 PROBLEMA IDENTIFICADO

O sistema mostra o aviso "Valor excede o saldo disponível", mas **ainda permite salvar a despesa**.

**Causa:** A validação `canSubmit` não verifica se há saldo disponível.

---

## 🔧 SOLUÇÃO EM 2 PASSOS

### PASSO 1: Modificar `DespesaFormBasicFields.jsx`

Adicionar callback para informar ao componente pai quando o valor excede o saldo.

**📍 Localização:** `src/components/despesa/DespesaFormBasicFields.jsx`

**🔧 Linha 16-23 (Props do componente) - ADICIONAR nova prop:**

```javascript
const DespesaFormBasicFields = ({
  formData,
  errors = {},
  emendaInfo = null,
  modoVisualizacao = false,
  handleInputChange,
  despesaParaEditar = null,
  onValorExcedeSaldo, // ✅ NOVA PROP ADICIONADA
}) => {
```

**🔧 Linhas 48-50 (Após calcular valorExcedeSaldo) - ADICIONAR useEffect:**

```javascript
const saldoDisponivel = calcularSaldoParaValidacao();
const valorNum = parseValorMonetario(formData.valor);
const valorExcedeSaldo = valorNum > saldoDisponivel;

// ✅ NOVO: Notificar componente pai quando valor excede saldo
React.useEffect(() => {
  if (onValorExcedeSaldo) {
    onValorExcedeSaldo(valorExcedeSaldo);
  }
}, [valorExcedeSaldo, onValorExcedeSaldo]);
```

**📍 Localização:** `src/components/DespesaForm.jsx`

## 📝 OBSERVAÇÕES TÉCNICAS

### Por que usar callback ao invés de Context?

- ✅ **Simplicidade:** Apenas 2 componentes envolvidos (pai e filho)
- ✅ **Performance:** Evita re-renders desnecessários
- ✅ **Manutenibilidade:** Fluxo de dados claro e direto

### Por que usar useEffect no filho?

- ✅ Garante que pai é notificado sempre que `valorExcedeSaldo` muda
- ✅ Evita prop drilling complexo
- ✅ Mantém lógica de validação encapsulada no componente filho

---

## 🐛 TROUBLESHOOTING

**Problema:** Botão continua habilitado mesmo com valor excedendo saldo

**Solução:** Verifique se:
1. `onValorExcedeSaldo` está sendo passada corretamente como prop
2. `setValorExcedeSaldo` está sendo chamado no useEffect
3. `!valorExcedeSaldo` foi adicionado na condição `canSubmit`

**Problema:** Botão fica desabilitado mesmo com valor correto

**Solução:** Verifique se:
1. `valorExcedeSaldo` está sendo atualizado para `false` quando valor é corrigido
2. useEffect tem as dependências corretas `[valorExcedeSaldo, onValorExcedeSaldo]`
