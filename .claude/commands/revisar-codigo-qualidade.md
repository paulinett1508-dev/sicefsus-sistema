# Code Review

Faça review do arquivo ou pasta especificado:

## Checklist
- [ ] Código limpo e legível
- [ ] Nomes de variáveis descritivos
- [ ] Funções com responsabilidade única
- [ ] Tratamento de erros adequado
- [ ] Sem código comentado/morto
- [ ] Sem console.log de debug
- [ ] Imports organizados

## Análise
1. Complexidade: identifique funções muito longas (>50 linhas)
2. Duplicação: encontre código repetido
3. Performance: identifique re-renders desnecessários

## Sugestões
Liste melhorias ordenadas por prioridade (alta/média/baixa)
