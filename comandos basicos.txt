🔄 COMANDOS ESSENCIAIS DO switch-env.sh

bash# 1. VERIFICAR AMBIENTE ATUAL
./switch-env.sh check
# Mostra se está em DEV ou PROD

# 2. MUDAR PARA DESENVOLVIMENTO
./switch-env.sh dev
# Conecta ao banco de testes (60dbd)

# 3. MUDAR PARA PRODUÇÃO
./switch-env.sh prod
# Pede confirmação "sim"
# Conecta ao banco de produção

# 4. VER STATUS COMPLETO
./switch-env.sh status
# Mostra todos os arquivos .env disponíveis

🎯 FLUXO DE USO DIÁRIO
bash# MANHÃ - Começar desenvolvimento
./switch-env.sh check        # Ver onde está
./switch-env.sh dev          # Garantir que está em DEV
npm run dev                  # Desenvolver

# DEPLOY - Quando tiver novidades prontas
./switch-env.sh prod         # Mudar para PROD
# Digite: sim                # Confirmar
npm run build:prod           # Build de produção
# Redeploy no Replit        # Deploy
./switch-env.sh dev          # VOLTAR PARA DEV!

# 3. TESTAR LOCALMENTE (opcional mas recomendado)
npm run preview

# VERIFICAÇÃO - A qualquer momento
./switch-env.sh check        # Onde estou?

⚠️ REGRA DE OURO
bash# SEMPRE após deploy
./switch-env.sh dev

# NUNCA deixe em prod após deploy!

💡 RESUMO VISUAL
check  → Ver ambiente atual
dev    → Modo desenvolvimento (banco teste)
prod   → Modo produção (banco real) 
status → Ver configuração completa
Dica: 99% do tempo use dev. Use prod só para deploy!

🚀 COMANDOS DISPONÍVEIS

🔧 BUILDS:

npm run build        # Sync + build (sem incremento)
npm run build:dev    # Build desenvolvimento (sem incremento)
npm run build:prod   # Build produção (COM incremento automático)

📝 VERSIONAMENTO MANUAL:
npm run version:patch   # 2.3.44 → 2.3.45
npm run version:minor   # 2.3.44 → 2.4.0
npm run version:major   # 2.3.44 → 3.0.0
npm run version:sync    # Só sincroniza arquivos

✅ CONFIRMAÇÃO DO FLUXO COMPLETO

🎯 PIPELINE DE PRODUÇÃO:
1. Desenvolvedor: npm run build:prod
   ↓
2. auto-increment.js: Detecta produção
   ↓
3. increment-version.js: 2.3.44 → 2.3.45
   ↓
4. generate-version.js: Sincroniza tudo
   ↓
5. vite build: Build com nova versão
   ↓
6. Deploy: Sistema atualizado
   ↓
